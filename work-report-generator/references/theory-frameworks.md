# 工作管理理论框架参考

## GTD (Getting Things Done)

**核心理念**：将所有任务从大脑中移出，放入外部系统管理，释放认知负担。

**五大步骤**：
1. **Capture（收集）**：将所有想法、任务、信息放入 Inbox
2. **Clarify（理清）**：判断每项任务是否可行动，可行动则定义下一步
3. **Organize（组织）**：按项目、优先级、时间归类
4. **Reflect（回顾）**：定期回顾系统，确保无遗漏
5. **Engage（执行）**：根据情境、时间、精力选择执行

**在日报中的应用**：
- `Inbox` 对应 Capture 步骤
- `Next Actions` 对应 Clarify + Organize
- `Done` 对应 Engage 的成果
- `Waiting` 对应需要他人协助的阻塞项

## STAR 法则

**核心理念**：用结构化方式描述工作成果，让成果可衡量、可理解。

**四个维度**：
- **S (Situation)**：工作背景和场景
- **T (Task)**：需要达成的目标和任务
- **A (Action)**：采取的具体行动步骤
- **R (Result)**：可量化的结果和产出

**在周报/月报中的应用**：
- 周报：按 STAR 格式提炼每项关键成果
- 月报（向上汇报）：STAR 格式让领导快速理解工作价值
- 特别适合测试工作：S=迭代背景，T=测试范围，A=测试执行，R=Bug数据/通过率

## PDCA 循环 (戴明环)

**核心理念**：持续改进的四个阶段循环。

**四个阶段**：
1. **Plan（计划）**：制定目标和行动计划
2. **Do（执行）**：按计划执行
3. **Check（检查）**：对比计划与实际，分析偏差
4. **Act（处理）**：根据检查结果改进，标准化成功经验

**在三级报告中的应用**：
- 日报：记录 Plan(明日计划) + Do(今日执行) + Check(偏差) + Act(改进)
- 周报：Check 聚合为周度偏差分析，Act 形成下周改进点
- 月报：月度总复盘，趋势分析，下月 Act 规划

**PDCA 在不同层级的关系**：
- 日报的 Act → 周报的 Plan
- 周报的 Act → 月报的 Plan
- 月报的 Act → 下月日报的 Plan（形成闭环）
