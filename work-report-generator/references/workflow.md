# 日循环工作流程

## 日循环：晨间规划 → 日间执行 → 晚间复盘

每日工作遵循以下循环：

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  🌅 晨间规划 → 🌞 日间执行 → 🌙 晚间日报           │
│                                                     │
│  晨间：读取昨日日报，生成今日规划                     │
│  日间：按规划执行工作                                │
│  晚间：记录今日成果，生成日报                         │
│                                                     │
│  └─ 晚间日报的"明日计划" → 明日晨间规划的"昨日遗留"  │
│     形成日循环闭环                                   │
│                                                     │
└─────────────────────────────────────────────────────┘
```

## 晨间规划生成流程

1. 确定日期（默认今天）
2. 查找昨日日报文件（`reports/YYYY/MM/daily/YYYY-MM-(DD-1).md`）
3. 若存在昨日日报，自动提取：
   - Next Actions 中未完成 → 今日待续任务
   - Done → 昨日完成回顾（提供上下文）
   - Waiting → 今日需跟进的阻塞事项
   - 明日计划 → 今日继承
4. 若不存在昨日日报，跳过自动提取环节
5. 询问用户今日新增任务：
   - "今天有新分配的任务吗？逐条描述（内容|来源|优先级|预计耗时）"
6. 合并昨日遗留 + 今日新增，按优先级排序，生成今日工作安排
7. 询问用户：
   - "对今日安排有调整吗？"
   - "今日注意事项？"
8. 读取模板 `assets/morning-plan-template.md`
9. 填充数据，生成文件到 `reports/YYYY/MM/daily/YYYY-MM-DD-plan.md`

## 晚间日报生成流程

晨间规划生成日报时，自动将晨间规划中的数据作为基础：

1. 确定日期（默认今天）
2. 查找今日晨间规划文件（`reports/YYYY/MM/daily/YYYY-MM-DD-plan.md`）
3. 若存在晨间规划，自动提取：
   - 今日工作安排 → 作为 Next Actions 的初始内容
   - 昨日遗留未完成 → 检查哪些今日已完成，转入 Done
   - 今日新增任务 → 作为 Inbox 的初始内容
4. 询问用户今日实际执行情况：
   - "今天实际完成了哪些任务？（对照晨间规划）"
   - "有什么阻塞事项？"
   - "专注度评分（1-5）？"
   - "偏差分析（计划 vs 实际）"
   - "改进措施？"
   - "明日计划？"
5. 读取模板 `assets/daily-report-template.md`
6. 填充数据，生成文件到 `reports/YYYY/MM/daily/YYYY-MM-DD.md`

## 日循环数据流转

```
昨日日报
  ├─ 未完成的 Next Actions → 今日晨间规划 "昨日未完成→今日待续"
  ├─ Waiting → 今日晨间规划 "昨日阻塞→今日跟进"
  ├─ 明日计划 → 今日晨间规划 "昨日计划→今日继承"
  │
今日晨间规划
  ├─ 今日工作安排 → 今日日报 Next Actions（基准）
  ├─ 今日新增 → 今日日报 Inbox
  │
今日日报
  ├─ Done → 周报 STAR 成果
  ├─ Waiting → 周报阻塞跟踪
  ├─ Check/Act → 周报 PDCA 复盘
  ├─ 明日计划 → 明日晨间规划（闭环）
```

# 三级报告工作流程

## 目录结构

所有报告文件存储在统一的目录结构中：

```
reports/
├── 2026/
│   ├── 01/                          # 1月
│   │   ├── daily/
│   │   │   ├── 2026-01-02.md        # 日报（跳过周末和节假日）
│   │   │   ├── 2026-01-03.md
│   │   │   └── ...
│   │   ├── weekly/
│   │   │   ├── 2026-W01.md          # 周报（第1周）
│   │   │   ├── 2026-W02.md
│   │   │   └── ...
│   │   └── 2026-01.md               # 月报
│   ├── 02/                          # 2月
│   │   └── ...
```

## 报告命名规范

| 类型 | 命名格式 | 示例 |
|------|---------|------|
| 晨间规划 | `YYYY-MM-DD-plan.md` | `2026-04-27-plan.md` |
| 日报 | `YYYY-MM-DD.md` | `2026-04-27.md` |
| 周报 | `YYYY-WXX.md` | `2026-W17.md` |
| 月报 | `YYYY-MM.md` | `2026-04.md` |

## 周编号规则

- 周编号使用 ISO 8601 标准（周一为一周开始）
- 每年第一周是包含1月4日的那一周
- 周报涵盖周一至周五的工作日

## 日报生成流程

1. 读取日报模板 `assets/daily-report-template.md`
2. 询问用户今日工作内容：
   - "今天有哪些新增任务？"
   - "今天计划执行什么？"
   - "今天完成了什么？"
   - "有什么阻塞事项？"
   - "专注度评分（1-5）？"
   - "明日计划？"
3. 填充模板，生成日报文件

## 周报生成流程

1. 找到本周所有日报文件（周一至周五）
2. 读取并汇总所有日报数据：
   - 合并所有 Inbox → 生成任务总览
   - 合并所有 Done → 篩选关键成果，按 STAR 格式重组
   - 合并所有 Waiting → 跟踪阻塞事项
   - 合并所有 Check/Act → PDCA 周度复盘
3. 询问用户：
   - "本周有哪些关键成果需要 STAR 格式展开？"
   - "偏差分析是否有补充？"
   - "下周计划？"
4. 填充周报模板，生成周报文件

## 月报生成流程

1. 找到本月所有周报文件
2. 读取并汇总所有周报数据：
   - 合并任务总览 → 月度统计
   - 合并所有 STAR 成果 → 按测试迭代版本分组重组
   - 合合所有偏差数据 → 月度趋势分析
3. 询问用户：
   - "本月参与了哪些迭代版本？"
   - "每个迭代的 Bug 数据和通过率？"
   - "月度亮点、不足、意外发现？"
   - "下月展望？"
4. 填充月报模板，生成月报文件

## 数据流转关系

```
日报(GTD为主)
  │
  ├─ Done → 周报(STAR成果提炼)
  ├─ Waiting → 周报(阻塞跟踪)
  ├─ Check/Act → 周报(PDCA复盘)
  │
周报(STAR+PDCA为主)
  │
  ├─ STAR成果 → 月报(按迭代分组)
  ├─ 偏差数据 → 月报(趋势分析)
  ├─ 统计数据 → 月报(月度统计)
  │
月报(STAR为主,向上汇报)
  │
  └─ Act改进 → 下月日报的Plan(闭环)
```
